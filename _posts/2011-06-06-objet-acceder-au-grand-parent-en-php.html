'Objet : accéder au grand parent en PHP' categories: - PHP - Ressources et tutos PHP tags: - orienté objet - php status: publish type: post published: true meta: _edit_last: '1' image: '' _syntaxhighlighter_encoded: '1' seo_follow: 'false' seo_noindex: 'false' --- S'il est courant de surcharger une méthode parente en PHP, il est plus rare d'accéder directement à la classe "grand -mère" sans passer par la mère. Pourtant... c'est possible, et voici comment :-) En effet, un héritage de PHP 4 permet préserver le contexte d'exécution de la classe fille quand on appelle une classe parente en utilisant son nom plutôt que l'opérateur "parent" (qui n'existait pas avant). Un exemple tout simple : {% highlight php %} class GrandMother { protected $_myVar = '5'; public function doSomething() { echo "GrandMother, value is " . $this->_myVar; } } class Mother extends GrandMother { public function doSomething() { echo 'Mother, value is '; // Never called } } {% endhighlight %} La classe fille n'utilise pas "parent", mais directement le nom de la classe grand-mère : {% highlight php %} class Child extends Mother { public function doSomething() { $this->_myVar = '20'; return GrandMother::doSomething(); } } {% endhighlight %} Il ne reste plus qu'à tester : {% highlight php %} $oObject = new Child; $oObject->doSomething(); // affiche "GrandMother, value is 20" {% endhighlight %} A utiliser avec précaution et modération ;-)
